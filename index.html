我的几点体会

  1. 错误要勇于面对：刚开始看到红色报错确实有些慌，但静下心来看报错信息，大多数问题其实都可以通过谷歌搜索或者看官方文档解决。
  2. 理解比复制更重要：很多时候照着教程一步步操作没问题，但一旦出错就容易手足无措。所以建议每一步操作都理解它在做什么，比如 Git 的身份信息设置，是为了让 Hexo 能正常提交代码到远程仓库。
  3. 配置要细心：Hexo 的配置文件 _config.yml 格式很严格，缩进、空格都不能出错，一定要小心修改。

总结

虽然只是一个简单的静态博客，但通过这次部署，我不仅熟悉了 Hexo 的基本使用，还学会了如何排查部署过程中的常见问题。这对我来说是学习新知的新起点。
